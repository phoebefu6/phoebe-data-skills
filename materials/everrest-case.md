# Everrest - case canon

Everrest is a fictional B2B2C retail platform used as the running case across
phoebe-data-skills. Brands and merchants sell to consumers through one
marketplace across Southeast Asia (SG, MY, ID, TH, VN, PH).

## Business model
- 400 merchants in 3 tiers (standard / premium / enterprise) across 8 retail categories.
- 20,000 registered consumers acquired via 6 channels (organic, paid search, social, email, referral, affiliate).
- Revenue = commission on completed orders; the platform team cares about
  merchant quality, category health, demand planning, and channel attribution.

## Schema (8 tables)
| Table | Grain | Key columns |
|---|---|---|
| merchants | 1 row per merchant | merchant_id, category, tier, joined_at |
| products | 1 row per SKU | product_id, merchant_id, category, price |
| customers | 1 row per consumer | customer_id, signup_date, channel_id, region |
| channels | 1 row per channel | channel_id, name |
| orders | 1 row per order | order_id, customer_id, merchant_id, order_ts, status |
| order_items | 1 row per line item | order_id, product_id, qty, unit_price, discount |
| payments | 1 row per payment | payment_id, order_id, method, amount, paid_ts |
| returns | 1 row per return | return_id, order_id, reason, return_ts |

## Named characters
- **M0007 "Summit Wholesale Co"** - enterprise Grocery merchant that is really a
  bulk wholesaler; order values ~59x platform median. The canonical outlier.

## Rules
- Everrest is the ONLY public case; never swap in client data.
- Data always generated by a seeded script (seed 42); never hand-edited.
- New skills reuse this schema and may add tables, never contradict existing ones.
